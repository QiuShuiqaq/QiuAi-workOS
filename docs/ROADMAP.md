# QiuAI WorkOS 版本规划

## 路线原则

QiuAI WorkOS 的版本规划围绕一个目标推进：

先稳定平台内核、商业权益和多端 API 契约，再扩展岗位模板和 AI 执行能力。

每个版本都要沉淀可复用资产：

- 领域模型
- API 契约
- 权限和权益模型
- 岗位模板
- 流程模板
- Agent 组件
- 工具连接器
- 行业知识结构
- KPI 和验收标准

## V0.1：Platform Kernel

目标：建立后续 PC 端、移动端、企业控制台和 AI 执行层都依赖的系统内核。

核心能力：

- Account、Tenant、Workspace
- Personal Workspace 和 Enterprise Workspace
- Plan、Subscription、Entitlement、UsageMeter
- Organization 和 Department 的企业版边界
- RoleTemplate、RoleInstance、Task、Artifact 的核心模型
- 统一错误格式和 API 契约草案
- 基础审计和成本记录结构

验收标准：

- 个人版和企业版使用同一套底层模型，而不是两套系统。
- 个人版不能创建部门、邀请成员或启用企业治理能力。
- 企业版可以以 Organization 为核心创建 Department、成员和岗位。
- 所有受限写操作都有 Entitlement 检查设计。

## V0.2：Enterprise Operating Core

目标：稳定企业版的组织、权限、审批、审计和岗位治理能力。

核心能力：

- 多租户隔离
- 用户、部门和权限
- 企业成员邀请
- 组织角色和数据范围
- 知识库绑定到岗位
- 工具连接器基础框架
- 人工审批节点
- 审计日志
- 成本预算
- 岗位 KPI 看板

验收标准：

- 企业可以把岗位分配到部门和负责人。
- 不同用户只能访问授权范围内的岗位、任务和产物。
- 关键任务可以进入人工审批。
- 管理员可以看到岗位的成本、效率和质量指标。

## V0.3：Execution Runtime Core

目标：建立可替换的 AI 执行层，而不是把系统绑定到单一 Workflow 或 Agent 框架。

核心能力：

- ExecutionRuntime Adapter
- Dify Adapter 或其他首个执行适配器
- WorkflowRun、AgentRun、Job 状态机
- 执行日志、失败重试和取消
- Artifact 生成和回写
- 模型网关基础
- 工具调用记录

验收标准：

- RoleInstance 可以通过统一接口触发底层执行。
- 执行层可以替换实现，不影响岗位、任务和企业治理模型。
- 每次执行都能追踪输入、输出、成本、日志和产物。

## V0.4：行业模板化

目标：把单客户交付能力沉淀为可复用行业岗位模板。

核心能力：

- RoleTemplate 版本管理
- 模板参数化配置
- 模板安装向导
- 默认知识结构
- 默认工具要求
- 默认 KPI 集合
- 模板复制与定制
- 模板发布流程

重点行业方向：

- 保健品行业：AI案例运营专员、AI客户回访专员、AI直播运营专员
- 制造业：AI售后专员、AI产品知识专员、AI出口资料专员
- 教育：AI招生专员、AI课程运营专员、AI教务专员
- 法律服务：AI合同审核专员、AI案例分析专员

验收标准：

- 完成一个客户项目后，可以抽象为模板并安装给第二个客户。
- 模板安装时只需要填写企业差异化配置。
- 模板升级不破坏企业已有岗位实例。

## V0.5：多端客户端基础

目标：为 Web 控制台、PC 端和移动端建立统一客户端架构。

核心能力：

- 统一 API SDK
- 共享领域类型
- Workspace 切换
- 个人版和企业版权益展示
- Web 企业控制台
- PC 端基础壳
- 移动端基础壳

验收标准：

- 多端通过同一套 API 契约访问后端。
- 权益判断来自服务端，不在客户端重复实现。
- 个人版和企业版在 UI 上有清晰但一致的工作空间模型。

## V0.6：高级定制模式

目标：为实施人员和开发者开放底层能力，同时保持普通用户界面简单。

核心能力：

- Workflow Designer
- Prompt 管理
- Agent 管理
- Tool 管理
- MCP 接入
- Memory 配置
- Human Approval 配置
- 测试运行和回放
- 版本对比

验收标准：

- 高级用户可以定制岗位内部执行逻辑。
- 普通企业用户仍然只看到岗位、任务、结果和绩效。
- 定制后的能力可以重新封装为模板资产。

## V1.0：企业数字劳动力平台

目标：形成可以正式商业化交付的企业平台。

核心能力：

- 企业组织架构
- 岗位模板库
- 岗位安装和运行
- 任务中心
- 审批中心
- 知识管理
- 工具连接器
- 模型管理
- 成本中心
- 审计日志
- 绩效看板
- 运营后台

验收标准：

- 可支持多个付费企业并行使用。
- 可持续沉淀和复用行业岗位模板。
- 平台能清晰回答每个 AI 岗位的产出、成本、质量和风险。

## V1.x：Marketplace

目标：让岗位模板成为可分发、可购买、可安装的平台资产。

核心能力：

- AI 岗位商城
- 模板上架和审核
- 模板购买和安装
- 模板评分和案例展示
- 模板计费
- 第三方模板发布者
- Marketplace 分成机制

验收标准：

- 企业可以一键安装岗位。
- 平台可以从模板交易和使用中获得收入。
- 第三方可以围绕行业岗位生态贡献模板。

## 长期方向

长期能力包括：

- 多岗位协同
- 人类员工与 AI 岗位混合协作
- 跨部门任务流转
- 岗位自动优化建议
- 基于执行数据的模板持续学习
- 企业数字劳动力编制规划
- AI 岗位绩效评估体系
